## Technical Implementation

**Core Focus:** Information retrieval research implementing similarity-based Retrieval-Augmented Generation (RAG) techniques

**Technology Stack:**
- **Languages:** Python, HTML
- **ML/NLP Libraries:** PyTorch, Transformers, Sentence Transformers
- **Vector Database:** Qdrant for similarity search
- **Infrastructure:** Docker, Docker Compose for containerization
- **Development:** Poetry for dependency management, Jupyter notebooks for experimentation

**Key Components:**
- **RAG System:** Complete implementation with document ingester, retriever, and vector store
- **Model Tuning:** Custom fine-tuning pipeline with adapter merging capabilities
- **AI Providers:** Multi-provider support (local and cloud)
- **Experiments:** Structured experiment framework for research reproducibility
- **CLI Tools:** Command-line interface for running experiments and queries

**Key Achievement:** Successfully trained and tested the model on personal hardware, demonstrating practical ML engineering skills beyond typical academic projects. This involved:
- Optimizing model training for limited GPU memory
- Implementing efficient data pipelines
- Creating reproducible experiment workflows
- Documenting research methodology and results

**Research Areas:** 
- RAG techniques and similarity-based retrieval
- Information retrieval systems evaluation
- Model fine-tuning and adapter merging
- Vector database integration and optimization
